後綴自動機(Suffix Automaton)

Posted on 2023-07-14 Edited on 2023-07-16

後綴自動機

好久以前寫的筆記…














時間複雜度：$O(nlogm)$ ，空間複雜度：$O(n)$ ，n為字串長度，m為字母大小(視為常數).

也可以以空間換取時間，以陣列取代transition的平衡二元樹，則時間複雜度變為 $O(n)$ ，空間複雜度變為 $O(nm)$ 。

後綴自動機可以做什麼?

Checking for occurence

Given a text $T$, and multiple patterns $P$. We have to check whether or not the strings in P appears as a substring of T.

This task can also be handled with other data structures such as AC-automaton and suffix array. We will describe the suffix automaton approach in $O(|T|+\sum_{}^{}{|P|})$ time.

We build a suffix automaton of the text t in $O(|T|)$ time. To check for a pattern $P_{i}$, we follow the trasitions, starting from $t_{0}$, according to the characters of $P$. If we can process the entire string $P$ this way, then the string appears in $T$；if at some point the transition does not exist, the string can not be found.

Side note : the algorithm also finds the length of the longest prefix of $P$ that appears in the text.

Number of different substrings

Given a string $S$. You want to compute the number of different substrings.

This can be computed by doing dp on the suffix automaton. However, there is a better way to compute this.
We can take advantage of the fact that each state $v$ matches to substrings of length $[minlen(v), len(v)]$. Given $minlen(v)=1+len(link(v))$, we have total distinct substrings at state $v$ being
\begin{align*}
len(v)-minlen(v)+1=len(v)-(1+len(link(v)))+1=len(v)-len(link(v)).
\end{align*}

Total length of all different substrings

Given a string $S$. We want to compute the total length of all its various substrings.

The way we compute this is a lot like the previous section. We simply compute the 等差級數和 between $[len(link(v))-1, len(v)]$ for all states.

This approach runs in $O(|S|)$ time, but experimentally (not by me) runs 20x faster than the memorized dp verison on randomized strings since it requires no extra space and no recursion.

Lexicographically $k$-th substring

Given a string $S$, and multiple queries. In each query, for a given number $K_{i}$, we have to fint the $K_{i}$-th string in the lexicographically ordered list of all substrings.

The solution of this problem is based on the dp idea of the previous two problems. We compute the number of distinct substrings starting from all states first. The lexicographically $k$-th substring corresponds to the lexicographically $k$-th path in the suffix automaton.

Smallest cyclic shift

Given a string $S$. We want to find the lexicographically smallest cyclic shift.

A classical problem that can be done by constructing a suffix array. We will see how we can trivally find the smallest cyclic shift by suffix automaton.
We construct a suffix automaton for the string $S+S$. Then the automaton will contain in itself as paths all the cyclic shifts of the string $S$.

From here, the problem is reduced to finding the lexicographically smallest path of length $|S|$. We simply start in the initial state and greedily pass through the transitions with the minimal character.

Total time complexity is $O(|S|)$.

Number of Occurences

For a given text $T$, answer multiple queries. For each given string $P$ find out the number of times $P$ appears in the string $T$ as a substring.

We construct the suffix automaton for the text $T$. Next, we do the following preprocssing: for each state $v$ in the automaton we calculate the number $cnt[v]$ that is equal to the size of the set $endpos(v)$. Then the answer for a given string is simply $cnt[t]$, where $t$ is the state corresponding to the string.

Next we consider how to compute $cnt[v]$.

For each state not created by cloning (excluding $t_{0}$), we initialize it with $cnt=1$. Then we go through all states in decreasing order of their length $len$, and add the current value $cnt[v]$ to the suffix links:
\begin{align*}
cnt[link(v)]+= cnt[v]
\end{align*}
Preprocessing takes $O(|T|)$ time, answering a query takes $O(|P|)$ time.

All occurrence positions

Given a text $T$, and multiple queries. For each given string $P$, display all starting positions of the occurrences of $P$ in the string $T$.

We compute the position $firstpos$ for all states in the automaton, i.e. for each state $v$ we want to find the position $firstpos[v]$ of the end of the first occurrence.

The positions $firstpos$ can be maintained throughout sa_extend(). When we create a new state $cur$, we set:
\begin{align*}
firstpos(cur) = len(cur) - 1
\end{align*}
When we clone a vertex $q$ as $clone$, we set:
\begin{align*}
firstpos(clone) = firstpos(q)
\end{align*}
Let $t$ be the state corresponding to a query string $P$. Clearly $firstpos(t)$ is part of the answer. What other states do we need to take into account? All states that correspond to strings for which $P$ is a suffix.

For each state, we save a list of suffix references leading to it. Then, we can compute for a state $t$ all ending positions by doing a DFS/BFS starting from $t$ using only the suffix references. Time complexity for handling a query is $O(answer(P))$, since we will not visit a state twice (only one suffix link leaves each state).

Note : Two different states can have the same $firstpos$ due to the existence of clones. This does not ruins the complexity, since each state can only have at most one clone. When we visit a cloned state, we simply do nothing and continue on our DFS/BFS.

Longest common substring of two strings

Given two strings $S$ and $T$, find the longest common substring.

We construct a suffix automaton for the string $S$.

Then, we take the string $T$, and for each prefix look for the longest suffix of this prefix in $S$.

We use two variables: the current state $v$, and the current length $l$, describing the current matching part.

Initially $v=t_{0}$ and $l=0$. Now we will see how we can add a letter $T[i]$ and recalculate the answer for it.

  • If there is a transition from $v$ with the character $T[i]$, we simply follow the transition and increase $l$ by one.
  • Else, we have to shorten the current matching part, i.e. by following the suffix link: $v=link(v)$, and shorten the length by assigning $l=len(v)$ simultaneously. We keep doing this until we find a transition or reach $t_{0}$.

    Finally, the answer to the task will be the maximum of all the values of $l$.

    Time complexity is $O(|S|+|T|)$, since in one move we can either increase $l$  by one, or make several passes through the suffix links, each one ends up reducing the value of $l$.

Problems

CF - Cyclical Quest

給定一個字串 $S$，與數筆詢問。對於每個給定的字串 $X$，輸出多少個 $S$ 的子字串與 $X$ cyclical isoomorphic。

Two strings are called cyclical isomorphic if one can rotate one string to get the other one.
$|S|<10^6，\sum_{}^{}{|T|}<10^6$

建一個 $S$ 的 SAM。對於每筆詢問，建一個字串 $T=X+X-(X末字元)$。這樣 $T$ 就包含所有與 $X$ cyclic isomorphic 的字串。接就可以在SAM上跑詢問了。

需要注意的是，一種與 $X$ cyclic isomorphic的字串可能會在 $T$ 中出現多次。解決的方式為對於每個state，紀錄是否已經計算過他的貢獻，如果已經計算過就不要再加一遍。
